## 연체경과일수 구간별 필수 안내 매트릭스

상담사는 연체경과일수에 따라 안내 항목과 톤을 달리해야 한다.

- **1~30일 (단기연체)**:
  - 안내 의무: 결제일 / 원금 / 연체이자 / 총 납부금액 / 납부기한 / 결제계좌
  - 톤: 안내 중심, 강한 압박 표현 금지
  - 신용 영향: 30일 초과 시 신용정보원 단기연체 정보로 등재 가능성 사전 고지
  - 분할납부: 가능 (사전 신청 시 익월 결제일 분할 적용)

- **31~60일 (중기연체)**:
  - 위 안내 + 거래정지 안내 / 신용평점 영향(이미 등재 시) 명시
  - 추가 안내: 채권추심법 제8조의2에 따른 정당한 변제 요청 절차 고지
  - 분할납부: 본인 신청 또는 채권관리팀 협의 후 가능
  - 추심 횟수: 1일 5회 이내 (채권추심법 제8조의5)

- **61일 이상 (장기연체)**:
  - 위 안내 + 채무조정 / 신용회복위원회 안내 / 법적 절차 가능성 안내
  - **금지**: 위협, 협박, 수치심 유발, 가족·직장 동의 없는 통보, 야간(21시~08시) 연락
  - 외부 채권추심 위탁 가능성 안내 (위탁 시 사전 통지 의무)

## 거래정지 / 블랙리스트 등재 기준

- **거래정지 (자체 시스템)**:
  - 트리거: 결제일 다음날(D+1)부터 자동 적용
  - 효과: 신규 결제 / 현금서비스 / 카드론 승인 차단 (가용한도와 무관)
  - 해제: 미납액 전액 입금 후 시스템 반영 (실시간~30분)
  - 안내 의무: 거래정지 상태 명시 + 가용한도가 있어도 결제 승인은 차단된다는 점 보충

- **단기연체 정보 (신용정보원 등재)**:
  - 등재 기준: **연체일 30일 경과 + 미납액 10만원 이상**
  - 등재 정보: 미납 일자 / 미납 금액 / 카드사명
  - 해소: 미납액 전액 변제 후 90일 경과 시 등재 정보 삭제 (변제 즉시 삭제 아님)
  - 통상 호칭: "블랙리스트 등재" — 정확한 명칭은 "신용정보원 단기연체정보"

- **장기연체 정보**:
  - 등재 기준: 연체일 90일 경과 + 미납액 100만원 이상
  - 신용평점 영향: 통상 50~150점 하락 (등재 사실만으로)
  - 회복: 변제 완료 후 5년 보존

## 채권추심법 제8조의2 — 정당 절차 고지 의무

채권추심자는 채무자에게 다음을 고지할 의무가 있다.

- **고지 4요소**:
  1. **사실 관계**: 미납 사실 / 카드번호 (마스킹) / 미납 금액 / 미납 일자
  2. **법적 근거**: "신용정보의 이용 및 보호에 관한 법률" / "채권의 공정한 추심에 관한 법률" 제8조의2
  3. **소속 / 신원**: 채권추심자 소속 (신한카드 컬렉션관리부 채권관리팀) / 성명 / 사번
  4. **민원 / 분쟁 방법**:
     - 자사: 신한카드 고객센터 1544-7000 (민원 접수)
     - 금융감독원: 1332 / e-금융민원센터 (https://www.fcsc.kr)
     - 신용회복위원회: 1600-5500 (채무조정 상담)
     - 한국소비자원: 1372 (소비자 분쟁 조정)

- **위반 시**: 금감원 행정처분 / 영업정지 / 임직원 징계 가능 (채권추심법 제17조)

## 채권추심법 제9조 — 금지 행위

다음 행위는 채권추심법 제9조에서 명시적으로 금지하며, 적발 시 즉시 0점 + critical_violation 트리거 대상이다.

- **금지 행위 유형**:
  - **폭행 / 협박 / 위협**: "법적 절차 진행" / "처벌받을 수 있다" 등 일방적 위협 발화
  - **공포 조성**: "큰일난다" / "어떻게 책임질 거냐" 등 막연한 공포 유발
  - **수치심 유발**: "부끄럽지 않으세요" / "본인이 자초하신 거" 등 인격 모욕
  - **거짓 표시**: 미확정 사실을 확정 사실처럼 안내 (예: "곧 구속됩니다")
  - **채무자 외 제3자 통보**: 가족 / 직장 / 친지에게 채무 사실 통보 (동의 없이)
  - **반복 연락**: 1일 5회 초과 / 야간(21시~08시) 연락
  - **불공정 추심**: 변제 능력 없음을 알면서 반복 청구

- **합법 응대 가이드**:
  - 사실 기반 안내 (감정 표현 배제)
  - 채무자 사정 청취 (분할납부 / 변제 시점 협의)
  - 거부/저항에는 \"채권추심법 절차에 따라 안내드리고 있습니다\" 멘트로 진정

## 가용한도 산정 기준

- **공식**: `가용한도 = 등록 한도 − 사용 잔액 − 미정산 매입액 − 한도 차감 약정금액`
  - 예: 등록 한도 500만원 / 사용 잔액 43만원 / 미정산 매입 0원 → 가용한도 457만원

- **연체 상태에서의 가용한도**:
  - 가용한도 산정 자체는 동일하게 표시
  - 단, **거래정지 적용 시 가용한도와 무관하게 신규 결제 승인 차단**
  - 안내 시 반드시 \"가용한도가 있어도 연체 해소 전 신규 승인 제한\" 보충 필수

- **한도 조정**:
  - 인상: 본인 신청 + 신용평가 통과 (연체 미해소 상태에서는 불가)
  - 인하: 본인 신청 즉시 적용 / 카드사 직권 인하 시 사전 통지
  - 안내 의무: 인하 시 사유 / 적용일 / 이의제기 방법 안내

## 분할납부 / 리볼빙 안내 절차

- **분할납부 유형**:
  - **즉시 분할**: 결제일 도래 후 미납액에 대해 신청 (월 단위 / 최대 24개월)
  - **사전 분할**: 결제 시점 또는 결제일 전에 신청 (할부 전환)
  - **리볼빙**: 약정 비율(5%~100%)만 결제하고 잔액은 익월 이월 (수수료 발생)

- **신청 방법**: 고객센터 1544-7000 / 신한플레이 앱 / 영업점
- **수수료**: 약 9.9%~19.9% (신용등급 / 잔여기간별 차등)
- **안내 의무**:
  - 수수료율 / 총 부담 금액 / 약정 기간 / 중도 상환 가능 여부
  - 본인 동의 절차 (구두 + 시스템 입력)
  - 약관 송부 (SMS 링크 또는 앱 알림)

## 연체 안내 5요소 표준 스크립트 (#901 직결)

상담사는 연체 안내 시 다음 5요소를 **누락 없이 + 숫자 정확히** 안내해야 한다. QA #901 정확한 안내 ★ 만점 기준.

- **5요소 체크리스트**:
  1. **연체경과일수**: "결제일로부터 OO일 경과되었습니다" — 채권시스템 원장과 일치
  2. **연체원금 + 수수료**: "원금 OOO원, 연체이자 OOO원" — 둘을 분리 명시 (합산만 안내 시 부분 점수)
  3. **총 납부금액**: "총 OOO원 납부하셔야 합니다" — 원금 + 이자 합계
  4. **납부기한**: "MM월 DD일까지" — 명확한 일자 (요일/주말 명시 권장)
  5. **결제계좌**: "OO은행 ***-**-**OOOO" — 마스킹 적용된 결제계좌
- **표준 발화 예시**:
  - "고객님 신한카드 ****-****-****-OOOO 번호로 N일 경과된 결제대금이 확인되고 있습니다."
  - "원금 OOO원이고, 여기에 연체이자 OOO원이 가산되어 현재 총 OOO원 납부하셔야 합니다."
  - "MM월 DD일까지 기존 결제계좌인 OO은행 ***-**-**OOOO 으로 납부해 주시면 연체이자 추가 가산 없이 마무리되실 수 있습니다."
- **복창 의무**: 고객이 5요소 중 하나를 다시 물어보면 그대로 복창 — 복창 누락 시 부분 점수 (15→10)
- **부분 점수 트리거**:
  - 5요소 중 1개 누락: 10점
  - 5요소 중 2개 누락 또는 1개 오안내: 5점
  - 3개 이상 누락 또는 오안내 미정정: 0점

## 오안내 즉시 정정 표준 (#901 부분 점수 회복)

상담 중 숫자/계좌/기한 등을 잘못 안내했을 때 **즉시 정정** 시 부분 점수 (5점) 인정. 정정 미시도 시 0점.

- **정정 트리거**:
  - 본인이 발화 직후 5턴 이내 인지
  - 또는 고객이 의문을 제기 ("어 그래요?", "확실해요?")
- **정정 표준 발화**:
  - "죄송합니다, 방금 안내 다시 드리겠습니다."
  - "정확히는 OOO원입니다. 혼동 드려 죄송합니다."
  - 정정 후 5요소 전체 재복창 권장 (정정 신뢰도 회복)
- **정정 미시도 시 critical_violation**:
  - 고객이 잘못된 정보로 납부 → 분쟁 / 민원 발생 시 카드사 책임
  - QA Layer 3 Override 입력에 `has_incorrect_guidance=True, correction_attempted=False` 로 마킹
- **정정 후 처리**:
  - 채권시스템 원장 재조회 (오안내 사실 기록)
  - 통화 종료 후 SMS / 앱 알림으로 정확 정보 재전송 (이중 안전망)

**source_ref**: shinhan-correction-protocol v1.0
