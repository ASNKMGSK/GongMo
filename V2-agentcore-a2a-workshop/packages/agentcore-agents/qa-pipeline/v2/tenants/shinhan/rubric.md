## 2. 주요 판정 원칙 (shinhan_v1)

- **#1 첫인사**: "안녕하세요 / 반갑습니다" + "신한카드" + 상담사명 3요소.
- **#2 끝인사**: "다른 문의사항 있으실까요" 추가 문의 확인 + 인사말/상담사명 중 1개만 있어도 3점 (2-요소 완화).
- **#3 경청**: STT 환경의 한계 (말겹침/말자름 식별 불가) 로 평가 제외.
- **#6 정중한 표현**: 반말/사물존칭 ("승인되시겠습니다", "입금되시겠습니다", "처리되시겠습니다") 1회 이상 시 감점.
- **#7 쿠션어**: refusal-gated — 한도 부족 / 발급 거절 / 청약철회 불가 등 거절 상황에서만 판정.
- **#8 문의 파악**: 의도 복창 누락 시 감점 (인바운드 핵심).
- **#9 / #17 / #18 force_t3**: 본인확인 / PII 누출은 confidence 무관 T3 강제 검수.
- **#10 설명 명확성**: 이자율 (연체이자 / 할부 수수료), 청구일, 영업일 (3~5일), 결제일 변경 가능 범위 등 숫자/조건 명시 여부.
- **PII strict 정책**: 주민번호 / 카드번호 전체 / 계좌번호 전체 마스킹 강제 (`pii_policy.strictness = strict`).
- **부서특화 정확도 (`*_accuracy`, 20점)**: 부서별 `business_knowledge/` 매뉴얼 RAG 필수. 매뉴얼 부재 시 `unevaluable` 반환.
- **법규 준수 노드**: 채권추심법(collection), 여전법/금소법(review/crm), 금소법(compliance) 위반 1회 → 0점.

## 3. 카테고리 전체 영향 (Critical)

- 카드번호 / 주민번호 / 계좌번호 마스킹 누락 → #17/#18 = 0 + 컴플라이언스 보고
- 반말 / 비하 / 욕설 / 협박성 발언 → 총점 0점
- 부당권유 / 설명의무 위반 (compliance) → `comp_unfair_sale_check` = 0 + 본부 이관
- 채권추심법 위반 (collection) → `coll_debt_compliance` = 0 + 본부 이관
- 매뉴얼 대비 허위 안내 미정정 (이자율/수수료 오안내) → 부서특화 `*_accuracy` = 0

## 4. 인프라 폴백 정화

LLM 실패 / ThrottlingException 기반 규칙 폴백은 `points=0` + `[SKIPPED_INFRA]` 태그. 감점 미반영.
