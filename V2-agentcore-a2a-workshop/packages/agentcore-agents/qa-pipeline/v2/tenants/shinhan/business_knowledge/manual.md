## 신한카드 상담 도입 의무사항

- **표준 도입 멘트** (3요소 필수):
  1. 인사말 — "안녕하세요" 또는 "반갑습니다"
  2. 소속 — "신한카드" (필요 시 부서명 — "신한카드 컬렉션관리부", "신한카드 고객센터" 등)
  3. 상담사명 — 본인 이름 (성+이름 풀네임 권장)
- **표준 예시**:
  - "안녕하세요, 신한카드 고객센터 ***입니다. 무엇을 도와드릴까요?"
  - "반갑습니다, 신한카드 컬렉션관리부 *** 사원입니다."
- **본인확인 선행**: 도입 직후 — 상담 의도 확인 → 본인확인 (BK-SHC-003) → 상세 상담 순서 준수.
- **녹취 안내**: 상담 품질 향상 및 분쟁 예방을 위해 모든 상담은 녹취. 첫 인입 시 자동 안내 메시지 또는 상담사 구두 안내.
- **상담사 정보 재고지**: 분쟁 / 민원 / 약관 설명 의무 발생 시 상담사명 / 사번 재고지.

## 신한카드 상담 종료 의무사항

- **표준 종료 멘트** (3요소 권장, 2요소 인정):
  1. 추가 문의 확인 — "다른 문의사항 있으실까요?" 또는 "더 도와드릴 부분 있으실까요?"
  2. 인사말 — "감사합니다" 또는 "이용해 주셔서 감사합니다"
  3. 상담사 재고지 — "신한카드 *** 사원이었습니다"
- **표준 예시**:
  - "다른 문의사항 있으실까요? 네 그럼 이용해 주셔서 감사합니다. 신한카드 *** 사원이었습니다."
- **민원 / 컴플레인 종료**: 처리 결과 / 다음 절차 / 회신 일정 명확히 안내 후 종료.
- **결제 / 청구 관련 종료**: 안내 내용 SMS / 알림톡 발송 여부 안내.
- **종료 부적절 케이스** (감점 대상):
  - 추가 문의 확인 누락
  - 일방적 종료 ("네 그럼 이만 끊겠습니다" 단독)
  - 인사말 누락

## 본인 확인 절차 (2-요소)

- **필수 확인 요소 (2-요소 조합)**:
  - **고객 성함** (풀네임)
  - 추가 1요소: 다음 중 하나
    - 생년월일 (6자리 또는 8자리)
    - 카드번호 일부 (앞 6자리 / 뒤 4자리)
    - 등록 휴대폰 번호 뒤 4자리
- **확인 절차**:
  - "본인확인 위해 성함과 생년월일 말씀 부탁드립니다."
  - 또는 "성함과 카드번호 뒤 4자리 부탁드립니다."
  - 고객 답변 후 "*** 고객님 본인 맞으십니까?" 재확인
  - "소중한 정보 확인 감사드립니다" 완료 멘트
- **STT 환경 마스킹 가정**:
  - 카드번호 / 주민번호 / 계좌번호는 `*` 또는 `***` 로 마스킹된 형태로 transcript 에 등장
  - 상담사가 카드번호 전체 / 주민번호 전체 를 **선발화** 하면 #17/#18 즉시 감점
- **본인 확인 불일치 시**:
  - 본인 외 가족 / 대리 통화: 위임 동의 또는 본인 재통화 요청
  - 개인정보 변경 (주소 / 연락처 등) 은 반드시 본인만 가능
- **금지 정보**:
  - 비밀번호 / CVC 번호 / 보안카드 번호 요청 금지
  - 주민등록번호 전체 노출 금지 (마스킹 후 일부 자리만 확인)

## 개인정보 취급 일반 원칙

- **마스킹 강제 (PII strict)**:
  - 카드번호 전체 16자리 노출 금지 — 앞 6자리 / 뒤 4자리만 확인
  - 주민등록번호 전체 13자리 노출 금지 — 생년월일 (앞 6자리) 만 확인
  - 계좌번호 전체 노출 금지 — 은행명 + 뒤 4자리만 확인
  - 위반 시 #17/#18 = 0 + 컴플라이언스 본부 이관
- **제3자 제공 금지**:
  - 가족 / 직장 동료 / 지인 등 제3자에게 고객 개인정보 (주소 / 연락처 / 카드번호 / 청구 내역 등) 제공 금지
  - 위임장 / 대리 동의 등 적법 절차 없이는 본인 외 제공 불가
- **상담사 선발화 금지 항목** (감점 대상):
  - 고객 주소 / 휴대폰 번호 / 카드번호 / 주민번호 / 계좌번호
  - 고객이 먼저 발화한 정보를 상담사가 복창하는 것은 허용 (단, 마스킹 일관성 유지)
- **추심 / 채권 관련 특례** (collection 부서):
  - 채권추심법상 가족 / 제3자에게 채무 사실 고지 금지
  - 위반 시 `coll_debt_compliance` = 0 + 본부 이관
- **녹취 보관**:
  - 상담 녹취는 관련 법령 (전자금융거래법 / 신용정보법 등) 에 따라 보관
  - 분쟁 / 민원 발생 시 녹취 사본 제공 가능 (본인 신청)

## 신한카드 일반 용어 사전

- **연체일 (DPD, Days Past Due)**: 결제일 다음날부터 입금일 전일까지의 경과일.
- **연체이자 (지연배상금)**: 연체 발생 시 결제 예정 금액에 적용되는 추가 이자 (회원약관 명시).
- **가용한도 (이용가능 한도)**: 총 한도 - 이용 잔액 (결제 예정 / 미결제 포함).
- **임시증액**: 일시적 결제 필요 시 한도 증액 신청. 1~3개월 단위, 심사 후 승인.
- **거래정지**: 분실 / 도난 / 연체 등으로 카드 사용이 중단된 상태. 신고 후 즉시 적용.
- **카드 재발급 (재발급 / 갱신)**:
  - 재발급: 분실 / 훼손 / 정보 변경으로 카드 신규 제작 (영업일 3~5일)
  - 갱신: 유효기간 만료 시 자동 / 신청 발급
- **결제일 변경**: 매월 결제일을 다른 날짜로 변경. 1~25일 중 선택. 변경 시 기존 청구분 처리 정책 별도 안내.
- **할부 거래**: 일시불 / 2~36개월 할부. 할부 수수료 (이자) 별도 적용.
- **청약철회**: 신용카드 회원가입 후 일정 기간 (개통일로부터 14일) 내 무조건 해지 가능. 금소법 준수.
- **부가서비스 (서비스 상품)**: 카드사 제공 부수 서비스 (보험 / 안심결제 / 포인트 등). 가입 / 해지 시 약관 고지 필수.
- **민원 (불만)**:
  - 1차 민원: 고객센터 접수 → 부서 처리
  - 금감원 민원: 외부 기관 접수 → 본부 이관 + 회신 의무
- **STT 마스킹 표기**: transcript 에서 카드번호 / 주민번호 등은 `*` 또는 `***` 로 표기됨. 평가 시 이 마스킹은 정상 처리.

## 부서 공통 RAG Fallback 안내

- **우선순위**:
  1. 부서별 매뉴얼 (`inbound/<dept>/business_knowledge/manual.md` 또는 `outbound/<dept>/...`)
  2. site-level 공통 매뉴얼 (본 파일 — `tenants/shinhan/business_knowledge/manual.md`)
  3. RAG 결과 미존재 시 `unevaluable` 반환
- **부서별 매뉴얼이 우선되는 영역**:
  - collection: 채권추심법 절차 / 분할납부 옵션 / 지급유예
  - review: 카드 심사 기준 / 발급 거절 사유 / 약관 고지 항목
  - crm: 마케팅 동의 / 캠페인 안내 / 청약철회 절차
  - consumer: 민원 분류 / 처리 일정 / 취약계층 정의
  - compliance: 금소법 / 부당권유 / 설명의무 항목
- **site-level 공통이 적용되는 영역** (부서 무관):
  - 도입 / 종료 멘트 표준 (BK-SHC-001, BK-SHC-002)
  - 본인확인 절차 (BK-SHC-003)
  - 개인정보 취급 일반 (BK-SHC-004)
  - 카드사 일반 용어 사전 (BK-SHC-005)
- **RAG retrieval 정책 (Dev4)**:
  - tenant_id 필터: `shinhan` (site-level) 와 `shinhan/<dept>` 모두 검색 후 점수 결합
  - 동점 시 부서 매뉴얼 우선
- **매뉴얼 갱신 주기**: 분기 1회 정기 / 법규 변경 시 즉시.

## 거절 / 제한 안내 시 쿠션어 사용 원칙
