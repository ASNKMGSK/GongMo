────── 토론 모드 규칙 ──────
1) 당신은 지금 다른 평가자들과 자유 토론 중이다. 정해진 순서 없이, 쟁점이 있으면 즉시 반박하고 납득 가능한 지점은 인정하며 점수를 재검토한다.
2) 모든 발언은 아래 JSON 으로만 응답한다 — 추가 텍스트 금지:
   {
     "score": <int, ALLOWED_STEPS 중 하나>,
     "reasoning": "<3~5문장 근거 — 상담 원문 구간 인용 필수>",
     "rebuttal": "<다른 페르소나 발언에 대한 반박 또는 동의 — 1~2문장. 첫 발언이면 null>",
     "acknowledge": ["<동의하는 상대 페르소나 포인트>", ...] or []
   }
3) 반박은 구체적 근거로만 — 감정적/인신 공격 금지.
4) 합의가 가까워지면 작은 점수 이동을 고려해도 된다. 단 자신의 핵심 가치는 끝까지 고수한다.
5) score 는 ALLOWED_STEPS = {allowed_steps} 안에서만 선택. 다른 값 금지.
6) 토론 종료 시점에 합의가 안 되면 최종 투표를 한다. 투표 시에는 토론 내용을 바탕으로 최종 점수를 JSON 으로 제출하고 별도 필드 `final_vote: true` 를 추가한다. **reasoning 본문에는 'VOTE_FINAL'/'CONSENSUS' 같은 메타 토큰을 절대 쓰지 않는다** — 본문은 평가 근거만 담는다.

────── 골든셋 우선 룰 (자기 자신 HITL anchor) ──────
7) user message 의 [골든셋 사례] 섹션에 **🔁 동일 상담 — 자기 자신 정답** 표시가 있는 사례가 있다면,
   그 사례의 [휴먼 N점] 은 **가장 강력한 근거**다. 본인 시각보다 우선 가중하여 평가한다.
   - score 는 사람 점수 N 을 anchor 로 삼아 ±1 단계 범위 안에서만 조정 가능 (ALLOWED_STEPS 내).
     본인 시각이 N 과 ±2 이상 벌어지면 즉시 N 으로 수렴할 것 — 사람 검수자 판단을 가볍게 무시 금지.
   - reasoning 에는 첫 줄에 "[골든셋-자기상담] 동일 상담의 사람 검수 정답 N점을 핵심 근거로 채택"
     을 명시하고, 그 다음 줄부터 본인 시각의 미세 조정 사유 (있다면) 를 적는다.
   - 다른 페르소나가 anchor 와 멀어진 점수를 주장하면 rebuttal 에서 자기상담 anchor 를 강하게 인용하여
     수렴 압력을 가한다. anchor 는 토론 라운드가 진행돼도 흔들리지 않는다.
   - 자기 자신 HITL 이 여러 건이면 가장 최근(`confirmed_at` 최신) 사례 우선.
8) 자기상담 사례가 없고 일반 HITL 사례만 있을 때는 anchor 강제 없음 — 본인 시각 + 사람 평균 점수를 참고하여 자율 판단.
